IF APP STILL CRASHES ON DEVICE - DO THIS:
================================================================================

STEP 1: Clear App Data on Device
  1. Go to Settings → Apps → Café Ordering App
  2. Tap "Storage and cache"
  3. Tap "Clear cache" and "Clear data"
  4. Uninstall the app

STEP 2: Rebuild and Reinstall
  Command:
  .\gradlew.bat clean assembleDebug

  Then install:
  adb uninstall com.example.projectyear
  adb install app/build/outputs/apk/debug/app-debug.apk

STEP 3: If Still Crashes - Enable Logging
  Attach device with USB Debugging enabled
  Run logcat:
  adb logcat | grep -i "cafe\|exception\|error"

  This will show the exact error causing the crash

================================================================================
IF YOU STILL GET CRASHES:
================================================================================

Common Issues & Solutions:

1. "Activity not found"
   → Check AndroidManifest.xml has all activities
   → Verify activity names match class names

2. "NullPointerException"
   → A view wasn't found in layout
   → Check layout file has correct view IDs
   → Verify setContentView() is called first

3. "Database error"
   → First launch - databases are created automatically
   → Clear app data and reinstall
   → Check User table has schema

4. "Permission denied"
   → Add permissions to AndroidManifest.xml
   → Grant permissions at runtime (if API 23+)

5. "ClassNotFoundException"
   → Check import statements
   → Verify class names are correct
   → Rebuild project
